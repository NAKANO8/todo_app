# Requirements Document

## Project Description (Input)
企業導入を検討する組織の管理者（情シス担当）は、複数人・複数組織で同じTodoアプリを安全に運用するための権限管理手段を必要としている。現状、`users`テーブルにはロールの概念が存在せず、認証済みの全ユーザーが同一の権限を持つため、管理者が特定ユーザーの権限を制御したりアカウントを無効化したりする手段がない。この導入では、`users`テーブルに`role`カラム（初期値は`admin`と`member`の2種類、`group_leader`はグループ機能実装時に追加）を追加し、マイグレーションを実装する。これは今後の管理者画面・認可（Authorization）・アカウント無効化機能の前提となる基礎部分である。

### 非対象（将来の拡張として考慮）
グループ/メンバー管理（`group_leader`ロール、read/manage権限分離、グループ単位のスコープ管理）は本スペックの対象外。ただし、グループ機能実装時に`group_leader`を後から追加できるよう、`role`カラムはブール値ではなく、値の追加に耐えられる型（enum等）で設計すること。

## Introduction
現在このアプリのユーザーアカウントには権限の区別が存在せず、全ユーザーが同一の操作権限を持つ。本機能は、アカウント管理機能に「ロール」という概念を導入し、`admin`と`member`という2種類の値でアカウントを区別できるようにする。これは、今後実装される管理者画面・認可（Authorization）・アカウント無効化機能が拠り所とする、権限管理の基礎部分である。

## Boundary Context (Optional)
- **In scope**: 全アカウントに対するロール値の保持、新規登録時のデフォルトロール割り当て、導入前から存在するアカウントへのロール付与、自身のロール情報の参照、ロール値の入力による不正な自己昇格の防止
- **Out of scope**: 管理者画面（ユーザー一覧・権限変更・アカウント無効化UI）、ロールに基づく認可（アクセス制御）の実装、初期管理者アカウントの作成フロー、`group_leader`ロールおよびグループ単位の権限管理
- **Adjacent expectations**: 本機能は今後の「管理者画面」「認可（Authorization）」「初期管理者アカウントの作成フロー」の各仕様が前提とする基礎データを提供する。それらの仕様は、ここで導入するロール値を読み取って動作するが、ロールの割り当て変更手段（管理者画面等）は別仕様の責務とする。

## Requirements

### Requirement 1: ロール値の保持
**Objective:** As a システム運用者, I want すべてのユーザーアカウントがロール値を持つようにしたい, so that 今後、権限に応じた機能提供や制御の基盤ができる

#### Acceptance Criteria
1. The アカウント管理機能 shall すべてのユーザーアカウントに対してロール値を保持する。
2. The アカウント管理機能 shall ロール値を現在サポートされる値の集合（`admin`、`member`）に制限する。
3. The アカウント管理機能 shall 既存のロールデータを変更することなく、将来新しいロール値を追加できるようにする。

### Requirement 2: 新規登録時のデフォルトロール割り当て
**Objective:** As a 新規登録者, I want 特別な操作をせずにアカウントを使い始めたい, so that 登録直後から一般利用者として通常通りアプリを利用できる

#### Acceptance Criteria
1. When 新規ユーザーが登録される, the アカウント管理機能 shall そのアカウントに`member`ロールを割り当てる。
2. If 登録リクエストにロール値が含まれている, then the アカウント管理機能 shall その値を無視し、常に`member`を割り当てる。

### Requirement 3: 導入前から存在するアカウントへのロール付与
**Objective:** As a 導入前から利用している既存ユーザー, I want 機能追加後も変わらずログインして使い続けたい, so that 本機能の導入によって既存の利用が妨げられない

#### Acceptance Criteria
1. When 本機能が導入される, the アカウント管理機能 shall 導入前から存在するすべてのアカウントに`member`ロールを割り当てる。
2. While 本機能の導入作業が行われている間, the アカウント管理機能 shall 既存アカウントの認証・ログイン機能を変更なく提供し続ける。

### Requirement 4: 自身のロール情報の参照
**Objective:** As an 認証済みユーザー, I want 自分のアカウント情報を取得する際に自分のロールも確認したい, so that 自分がどの権限で利用しているかを把握できる

#### Acceptance Criteria
1. When 認証済みユーザーが自分のアカウント情報を取得する, the アカウント管理機能 shall そのレスポンスに現在のロール値を含める。
