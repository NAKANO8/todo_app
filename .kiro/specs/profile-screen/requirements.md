# Requirements Document

## Project Description (Input)
プロフィール画面の最低限実装。ログイン中ユーザーが自分の識別情報を確認し、パスワードを変更できる画面を追加する。アバター画像アップロードは工数が重くなるため初期実装では見送り、UI改善フェーズ以降に回す。

## Boundary Context
- **In scope**: ログイン中ユーザー自身のメールアドレス表示、表示名(display name)の設定・変更、パスワードの変更(現在パスワードの確認、新パスワードの強度チェック、変更成功時の他セッション強制ログアウトを含む)
- **Out of scope**: アバター画像アップロード、メールアドレスの変更、アカウント削除、メールアドレスの確認(メール認証)、パスワードを忘れた場合の再設定(パスワードリセット)、新規登録フローの変更、他ユーザーのプロフィール参照・編集、既存の管理者向けユーザー一覧画面の表示項目変更
- **Adjacent expectations**: 本機能は、既存のセッションベース認証および未認証リクエストを拒否する仕組みにそのまま乗ることを前提とする。パスワード変更成功時に他セッションを無効化する能力は、既に提供されている強制セッション無効化の仕組みを利用することを期待するが、具体的な連携方法は設計フェーズで定める。

## Requirements

### Requirement 1: プロフィール情報の表示
**Objective:** As a ログイン中ユーザー, I want 自分のメールアドレスと表示名を確認したい, so that 自分がどのアカウントでログインしているかを把握できる

#### Acceptance Criteria
1. When ログイン中ユーザーがプロフィール画面を開く, the プロフィール機能 shall そのユーザーのメールアドレスを表示する
2. While 表示名が設定されている, the プロフィール機能 shall その表示名を表示する
3. While 表示名が未設定である, the プロフィール機能 shall 表示名が未設定であることが分かる表示を行う
4. If 未認証のリクエストでプロフィール画面へのアクセスが試みられる, then the プロフィール機能 shall アクセスを拒否する

### Requirement 2: 表示名の設定・変更
**Objective:** As a ログイン中ユーザー, I want 自分の表示名を設定・変更したい, so that メールアドレスだけでなく分かりやすい表示で自分のアカウントを認識できる

#### Acceptance Criteria
1. When ログイン中ユーザーが新しい表示名を入力して保存を要求する, the プロフィール機能 shall その表示名でユーザーの表示名を更新する
2. If 入力された表示名が空文字である, then the プロフィール機能 shall 更新を拒否し理由を提示する
3. If 入力された表示名が50文字を超える, then the プロフィール機能 shall 更新を拒否し理由を提示する
4. When 表示名の更新が成功する, the プロフィール機能 shall 更新後の表示名をその場で画面に反映する

### Requirement 3: パスワードの変更
**Objective:** As a ログイン中ユーザー, I want 自分のパスワードを変更したい, so that アカウントのセキュリティを自分の意思で維持できる

#### Acceptance Criteria
1. When ログイン中ユーザーが現在のパスワードと新しいパスワードを入力して変更を要求する, the プロフィール機能 shall 現在のパスワードが正しいことを確認した上でパスワードを更新する
2. If 入力された現在のパスワードが実際のパスワードと一致しない, then the プロフィール機能 shall パスワードの変更を拒否し理由を提示する
3. If 新しいパスワードが登録時と同じ強度要件(8文字以上、大文字1文字以上、数字1文字以上を含む)を満たさない, then the プロフィール機能 shall パスワードの変更を拒否し理由を提示する
4. When パスワードの変更が成功する, the プロフィール機能 shall 変更操作を行ったセッション以外の、そのユーザーの既存セッションをすべて無効化する
5. When パスワードの変更が成功する, the プロフィール機能 shall 変更操作を行った本人の現在のセッションを有効なまま維持する

### Requirement 4: アクセス制御
**Objective:** As a システム運用者, I want プロフィール情報の参照・変更をアカウント本人のみに限定したい, so that 他人のアカウント情報が不正に閲覧・変更されることを防げる

#### Acceptance Criteria
1. The プロフィール機能 shall ログイン中ユーザー自身の情報のみを参照・変更の対象とする
2. The プロフィール機能 shall 他ユーザーのメールアドレス・表示名・パスワードを参照・変更する手段を提供しない
